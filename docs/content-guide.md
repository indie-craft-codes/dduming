# 뚜밍(dduming) — 콘텐츠 작성 가이드

글은 `posts/{slug}.json` 파일 하나로 작성한다. `git push`하면 빌드가 정적 HTML + SEO를 자동 생성해 배포한다. SEO뿐 아니라 GEO(생성형 AI 검색 최적화)를 함께 고려한다.

## 1. 글 = JSON 한 파일

| 필드 | 설명 |
|--|--|
| `slug` | URL·파일명·PK (불변) |
| `category` | 기술 / 경제 / 부동산 |
| `tags` | 태그 배열 |
| `title` | 제목, 25자 내외, 숫자·키워드 포함 권장 |
| `deck` | 부제 한 문장, 60자 내외 |
| `answerBox` | 핵심 요약 40~50단어 (결론 먼저) |
| `datePublished` / `dateModified` | 작성일 / 수정일 |
| `status` | `published` \| `draft` |
| `coverImage` | (선택) 없으면 커버 자동생성 |
| `blocks[]` | 본문 (아래) |

**자동 파생**(안 써도 됨): `toc`(h2에서), `readingMinutes`, `index.json`, 커버 이미지. 자세히는 [`../posts/README.md`](../posts/README.md).

## 2. 본문 = blocks[] 배열

블록을 순서대로 나열한다. 타입 목록·매핑은 `posts/README.md`. 계층은 H2(목차 앵커) → H3 → 문단 순.

- H2에는 `id`를 주면 목차·앵커가 자동 생성된다. 질문형 소제목 권장("~은 무엇인가?").
- 각 H2 섹션은 3~5문단 이내로 유지(AI가 문단 추출하기 좋게).
- 표·차트·체크리스트·콜아웃·FAQ·이미지 갤러리 등 블록으로 근거를 붙인다.
- 계산기 같은 인터랙티브 위젯은 컴포넌트 블록(예: `{ "type": "giftTaxCalc" }`).

## 3. 핵심 요약(answerBox)

본문 맨 위에 자동 배치된다. AI 검색엔진이 이 문단을 그대로 인용하는 경우가 많다.

- 40~50단어(한글 약 90~120자)
- "왜 / 무엇을 / 어떻게"의 결론을 먼저
- 숫자·고유명사 포함, 모호한 일반론 금지

## 4. SEO / GEO — 빌드가 자동

작성자는 `title`·`deck`·`tags`·날짜만 정확히 채우면 된다. 빌드가 자동 생성:

- JSON-LD `Article` 스키마 · Open Graph/Twitter 카드 · canonical
- `sitemap.xml` · `robots.txt` · `feed.xml`(RSS)
- H1→H2→H3 계층 · H2 앵커 · 목차

## 5. 금액 표기 규칙 (통일)

| 위치 | 형식 | 예 |
|--|--|--|
| 본문·표 | **억/만원** (읽기 쉽게) | 7억원, 6억 5,000만원, 1억 3,095만원 |
| 계산기·도구 | **원** (정밀값) | 130,950,000원 |

## 6. 자주 하는 실수

- 핵심 요약 없이 바로 본문 시작 — AI가 인용할 문단이 없어짐
- H2를 건너뛰고 H3부터 사용 — 계층이 깨져 파싱 어려움
- 숫자·출처 없이 뭉뚱그린 주장 — 표·차트로 뒷받침
- 금액 형식 혼용 — 위 5번 규칙을 따를 것

# 브리프 블로그 — AI 콘텐츠 작성 가이드

AI로 글을 쓸 때 아래 구조와 순서를 따르면 블로그 디자인(Blog Post 템플릿)에 그대로 맞습니다. SEO뿐 아니라 GEO(생성형 AI 검색엔진 최적화)를 함께 고려한 가이드입니다.

## 1. 글 구조 순서

항상 이 순서로 작성하세요:

1. **제목 (H1)** — 핵심 키워드 포함, 25자 내외
2. **부제 (deck)** — 한 문장 요약, 60자 내외
3. **메타 정보** — 작성일, 수정일, 예상 읽기 시간
4. **핵심 요약 (Answer Box)** — 40~50단어로 결론 먼저 (3번 항목 참고)
5. **목차 (TOC)** — H2 소제목 목록, 각 항목이 본문 앵커로 연결
6. **본문** — H2 대주제 → H3 소주제 순으로 계층화
7. **태그 · 관련 글** — 마지막에 배치

## 2. 제목 계층 규칙 (SEO/GEO 핵심)

| 태그 | 용도 | 작성 팁 |
|---|---|---|
| H1 | 글 제목, 1개만 | 숫자·키워드 포함 권장 |
| H2 | 대주제, 목차와 1:1 매칭 | 질문형 소제목 권장 ("~은 무엇인가?") |
| H3 | 세부 항목, 블록 제목 | 체크리스트·표 등 블록 바로 위에 배치 |

- AI가 문단을 추출하기 쉽도록, 각 H2 섹션은 3~5문단 이내로 유지
- H2를 건너뛰고 H3부터 쓰지 않기 — 계층이 깨지면 파싱이 어려워짐

## 3. 목차(TOC)와 앵커

본문의 모든 H2에는 `id="sec-키워드"`를 붙이고, 목차 링크는 `href="#sec-키워드"`로 연결합니다.

예: "1. 파운드리, 더 이상 병목이 아니다" → `id="sec-foundry"`

목차 문구와 실제 소제목 문구는 정확히 일치해야 합니다.

## 4. 핵심 요약(Answer Box) 작성법

본문 최상단, 목차 위에 배치합니다. AI 검색엔진이 이 문단을 그대로 인용하는 경우가 많습니다.

- 40~50단어(한글 기준 약 90~120자) 분량
- "왜 / 무엇을 / 어떻게"에 대한 결론을 먼저 제시 — 본문에서 풀어낼 논리를 미리 요약
- 숫자·고유명사를 포함해 구체적으로 작성 (모호한 일반론 금지)

## 5. 스키마 마크업 (JSON-LD)

모든 게시글에 아래 필드를 채워 `Article` 스키마로 발행합니다 — AI 검색엔진과 포털이 글의 종류·저자·날짜를 정확히 인식합니다.

```
headline, description, datePublished, dateModified,
author, publisher, articleSection, keywords, inLanguage
```

## 6. 주제별 추천 콘텐츠 블록

템플릿에 이미 준비된 블록들입니다. AI에게 "이 블록으로 작성해줘"라고 지정하면 됩니다.

| 블록 | IT/기술 글 | 부동산/경제 글 |
|---|---|---|
| 막대·선·도넛·가로막대 그래프 | 가동률, 벤치마크 추이 | 지수 추이, 지역별 비중 |
| 표 | 스펙·가격 비교 | 지역별 분양가·전세가 비교 |
| 코드 블록 | 스니펫, 설정 예시 | (거의 사용 안 함) |
| 체크리스트 | 배포 전 점검 항목 | 계약 전 확인 사항 |
| 번호·불릿 목록 | 진행 순서, 체크포인트 | 절차, 이번 주 이슈 |
| 콜아웃 | 주의사항, 참고 | 면책·출처 고지 |
| 이미지 갤러리(2단) | 스크린샷, 다이어그램 | 현장 사진, 평면도 |
| 영상 임베드 | 데모, 튜토리얼 | 현장 브리핑 |
| FAQ 아코디언 | 자주 묻는 기술 질문 | 계약·세금·절차 질문 |

## 7. AI에게 줄 프롬프트 예시

> 부동산 카테고리 글을 써줘. 제목/부제/메타/핵심요약(45단어)/목차(H2 4개)/본문(H2마다 3문단, 관련 있으면 H3에 표나 체크리스트 포함)/태그 5개 순서로, 정중한 문체로 작성해줘. 각 H2는 질문형으로.

## 8. 자주 하는 실수

- 핵심 요약 없이 바로 본문 시작 — AI가 인용할 문단이 없어짐
- H2를 건너뛰고 H3부터 사용 — 계층 구조가 깨져 파싱이 어려워짐
- 목차 항목과 실제 소제목 문구 불일치
- 숫자·출처 없이 뭉뚱그린 주장 — 표나 그래프로 뒷받침할 것

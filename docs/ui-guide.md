# 브리프 블로그 — UI 디자인 가이드

이 문서는 디자인 시스템 요약입니다. 개발자(또는 Claude Code)에게 전달하면 동일한 톤으로 화면을 만들 수 있습니다.

## 1. 템플릿 2종

- **미니멀 (Minimal)** — 기본. 화이트/오프화이트 배경, 얇은 라인 구분, 산세리프 위주. IT/경제 뉴스에 어울리는 담백한 톤.
- **매거진 (Magazine)** — 프리미엄 매거진 톤. 크림 배경, 세리프 제목, 굵은 룰선, 큰 커버 이미지 오버레이.

`Blog Home.dc.html` 하나의 파일에서 Tweaks로 두 템플릿을 전환할 수 있습니다 (`variant: minimal | magazine`).

## 2. 컬러 시스템

포인트 컬러 1개만 바꾸면 배경 톤·구분선·태그·링크 색이 전부 같이 움직이도록 설계되어 있습니다 (`color-mix` 기반 자동 파생).

| 토큰 | 값 (기본) | 용도 |
|---|---|---|
| 포인트 컬러 | `#3b6ef5` (파랑) | 태그, 링크, 강조, 그래프 |
| 색상 톤 | pastel / muted / vivid | 포인트 컬러의 채도·강도 조절 |
| 잉크(텍스트) | `#18181b` | 본문 제목·기본 텍스트 |
| 서브텍스트 | `#6b6b70` | 메타정보, 캡션, 보조 설명 |
| 배경 | `#faf9f6` + 포인트 5% 혼합 | 페이지 배경 |
| 구분선 | `rgba(24,24,27,.12)` + 포인트 혼합 | 카드 경계, 섹션 구분 |
| 성공/상승 | `#1a7a4c` (초록) | 경제 태그, 상승 지표 |
| 주의/하락 | `#b5452e` / `#b5680c` (주황) | 부동산 태그, 하락 지표 |

색상 톤 3단계: **파스텔**(부드럽게, 기본값) / **뮤트**(차분하게) / **비비드**(선명하게). 포인트 컬러 자체를 바꿔도(파랑→버건디 등) 이 톤 로직이 동일하게 적용됩니다.

## 3. 타이포그래피

한글·영문 모두 지원하는 무료 구글 폰트로 전체 통일:

- **본문/미니멀 제목**: IBM Plex Sans KR (400/500/600/700)
- **매거진 제목**: Noto Serif KR (500/600/700)
- **코드/라벨**: IBM Plex Mono (400/500)

크기 기준(데스크톱 / 모바일):
- H1: 40px / 26px
- H2: 25px / 21px
- H3: 19px / 17px
- 본문: 17px / 15.5px, line-height 1.8
- 메타/캡션: 12.5~13px

## 4. 레이아웃 & 반응형

- 최대 폭 1240px, 중앙 정렬
- 브레이크포인트: 760px 미만 = 모바일. 그 위는 데스크톱 (별도 태블릿 단 없음, 유동적으로 줄어듦)
- 그리드는 flex 기반 — 데스크톱은 좌우 배치, 모바일은 세로 스택으로 자동 전환
- 이미지는 고정 비율 프레임(16:10, 4:3, 21:9 등)에 `cover` 방식으로 꽉 채움 → 실제 이미지 크기가 달라도 그리드 정렬 유지
- 카드 전체 영역이 클릭 가능한 링크 (이미지+제목 개별 링크 아님)

## 5. 컴포넌트 목록

- 헤더: 로고 + 상단 내비게이션 (전체/기술/경제/부동산/소개)
- 홈: 히어로 카드 → 2단 그리드(대형+리스트) → 3단 카드
- 게시글 상세: 핵심요약 박스 → 목차(TOC, 앵커 이동) → 본문(H2/H3) → 그래프 4종(막대/선/도넛/가로막대) → 표 → 코드블록 → 체크리스트 → 목록 → 콜아웃 → 이미지 갤러리 → 영상 임베드 → FAQ 아코디언 → 태그 → 관련 글
- 카테고리 페이지: 필터 탭 → 대형+리스트 그리드 → 3단 카드 → 페이지네이션
- 소개(About) 페이지: 인트로(사진+텍스트) → 다루는 주제 3열 → 연락처

## 6. Tweaks (편집 가능 옵션)

`Blog Home.dc.html`에 4가지 트윅이 있습니다:

- **템플릿**: minimal / magazine
- **포인트 컬러**: 색상 자유 선택
- **색상 톤**: pastel / muted / vivid
- **모서리 모양**: sharp(각짐) / rounded(둥근)

## 7. SEO / GEO 고려사항 (게시글 상세)

- 제목 계층 H1→H2→H3, 질문형 소제목 권장
- 본문 서두 40~50단어 핵심 요약(Answer Box) 배치
- JSON-LD Article 스키마 (제목/설명/작성일/수정일/저자/카테고리/키워드)
- 각 H2에 `id="sec-키워드"` 부여, 목차와 앵커 링크
- 자세한 작성 규칙은 `content-guide.md` 참고

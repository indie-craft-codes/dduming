# 뚜밍(dduming) — UI / 디자인 가이드

프로덕션 화면은 `build/`가 `posts/*.json`을 정적 HTML로 렌더해 만든다. 스타일 단일 소스는 **`build/theme.mjs`**(디자인 토큰 + CSS). 아래는 그 요약이며, 값이 어긋나면 `theme.mjs`가 기준이다.

## 1. 컬러 시스템

포인트 컬러 1개 기준으로 배경·구분선·태그가 `color-mix`로 파생된다.

| 토큰 | 값 | 용도 |
|--|--|--|
| 포인트 | `#3b6ef5` (파랑) | 링크·강조·그래프·계산기 |
| 잉크(텍스트) | `#18181b` | 제목·본문 |
| 서브텍스트 | `#6b6b70` | 메타·캡션 |
| 배경 | `#faf9f6` + 포인트 4% | 페이지 |
| 구분선 | `rgba(24,24,27,.12)` | 카드·섹션 |

**카테고리 뱃지색**: 기술 = 파랑 `TECH` / 경제 = 초록 `#1a7a4c` `ECONOMY` / 부동산 = 주황 `#b5680c` `REAL ESTATE`.

## 2. 타이포그래피

- 본문·제목: **IBM Plex Sans KR** (400/500/600/700)
- 코드·라벨: **IBM Plex Mono**
- 크기(데스크톱/모바일): H1 44/28 · H2 25/21 · H3 19/17 · 본문 17/15.5(line-height 1.8) · 메타 12.5~13

## 3. 레이아웃 & 반응형

- 최대 폭 1240px, 본문(article) 760px 중앙 정렬
- 브레이크포인트: **760px 미만 = 모바일** (flex/grid가 세로 스택으로 전환)
- 이미지는 고정 비율 프레임(커버 16:10)에 `cover`
- **표는 `.tablewrap`으로 감싸 좌우 스크롤**(셀 `white-space:nowrap`)

## 4. 컴포넌트 (빌드 렌더)

- 헤더: 로고(뚜밍.) + 네비(전체/기술/경제/부동산/소개)
- 홈: 히어로 카드 + 3단 그리드
- 글 상세: **계산기 등 상단 위젯** → 핵심요약 → 목차 → 본문(블록) → 태그 → 관련 글
  - 커버 이미지는 **카드 썸네일·OG(소셜 공유) 전용**. 제목이 H1과 중복되어 본문에는 넣지 않음
- 카테고리 / About
- **JS 정책**: 차트=CSS·SVG, FAQ=native `<details>` → **zero-JS**. 계산기 등 진짜 인터랙션만 island(`build/components/`)
- 금액 표기: 본문·표 = 억/만원, 계산기·도구 = 원 (`content-guide.md` 5번)

## 5. 파일 구성

| 파일 | 용도 |
|--|--|
| `build/theme.mjs` | 디자인 토큰 + 공용 CSS |
| `build/blocks.mjs` | 블록 → HTML |
| `build/cover.mjs` | 커버 자동생성(SVG) |
| `build/components/` | 인터랙티브 컴포넌트(계산기 등) + 레지스트리 |
| `build/build.mjs` | 오케스트레이터(페이지·SEO·index 생성) |
| `templates/*.dc.html` | 초기 디자인 참고용(프로덕션 미사용) |

## 6. 참고

- 초기 원본 키트는 미니멀/매거진 2종·Tweaks를 가졌으나, 현 빌드는 **미니멀 톤 단일**로 구현.
- 테마 변경은 `theme.mjs` 한 곳만 수정하면 전체 반영(빌드가 캐시 해시로 파일명 갱신).
